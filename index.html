My Microgame

In October 2025 heb ik met mijn groepje een microgame gemaakt. We kozen ervoor om een idle game te maken met als inspiratie Idle Miner en Cash Inc. We hadden ervoor gekozen om als thema wiet/drugs verkoop te doen, je verbouwd wiet en verkoopt het om geld te verdienen. Hoe meer geld je krijgt, hoe meer upgrades je kan kopen, en hoe meer geld je verdient. Uiteindelijk kan je het spel verder uitbreiden door nieuwe kamers te kopen of nieuwe gebouwen om meer te wiet te planten. Het spel moest gespeeld worden op een telefoon. Dit was in het begin best lastig omdat we met sommige dingen niet goed wisten hoe het op telefoon werkte. Dit heeft er voor mij ook voor gezorgd dat ik veel vertraging had omdat ik soms vergat de instellingen goed in te stellen. Hierdoor heb ik een paar keer iets gemaakt wat ik later moest vervangen door een versie die wel werkte.

Wat ik heb gemaakt:

Samengevat: Ik heb flappy bird en achievement systemen gemaakt

  • Flappy bird: Complete werkende versie van flappy bird gemaakt. Helaas de eerste keer nadat ik hem gemaakt had ging hij stuk omdat ik een andere profile moest gebruiken om op te bouwen. Daarna heb ik opnieuw een flappy bird gemaakt met een ander input systeem, hier is veel tijd in gaan zitten omdat ik het weer opnieuw moest doen.
  • Achievements Aan het begin van het project was het eerste wat ik heb gemaakt heb een achievement systeem. Ik wist nog niet helemaal op welke manier ik dit zou gaan doen, dus ik heb veel dingen geprobeerd die niet werkten. Ik weet nu wel dat als ik een systeem wil maken met veel verschillende details en functionaliteiten, dat ik het beste een list kan gebruiken met een foreach loop. Ik heb eerst geprobeerd om dit met een class te doen maar dat was te veel werk en nog te ingewikkeld voor iets simpels. Ik kwam er dus achter dat een list goed werkte en heb zo een lijstje met achievements gemaakt.

Reflectie: Ik heb veel geleerd van het maken van deze systemen, mijn logica is weer iets bijgewerkt en ik vond het leuk om te expirimenteren met funcites van unity die ik zelf niet snel zou gebruiken. Wel vind ik het jammer dat ik er zo lang over heb gedaan, ik had graag gewilt dat ik meer had kunnen doen aangezien de rest van het groepje meer belangrijke dingen hebben gemaakt.
